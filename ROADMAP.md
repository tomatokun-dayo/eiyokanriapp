## 現在の状態

静的HTML/CSS/JSのプロトタイプ。モックから実用最小限へ移行中。

実装済み:

- 1食あたりの食材入力（行は可変。＋追加 / ×削除、初期3行）
- 入力単位は g / 小さじ / 大さじ（換算は共通値: 小さじ5g・大さじ15g）
- 食材マスタは外部ファイル化済み（10品、`state` / `spoonGrams` / `source` 付き）
  - 100gあたり栄養値は文科省 食品成分DBと照合済み（食品番号・参照URL保持）
- 月齢別ラインは外部ファイル化済み（値は仮のまま）
- localStorage 保存（`eiyokanri.entries.v1`）
  - 今日ビュー（合計・ログ・器・件数）は当日分のみ表示
  - リセットは当日分のみ削除
  - 7日間グラフは保存済み実データを日別集計
- 不足栄養素の提示（目安の70%未満を最大3つ、補う食材候補つき）

## 設計方針（明文化）

- **実行形態**: ビルドツールなし・ESモジュール不使用・`file://` 直開きで動くことを維持する。外部化は `<script defer>` の読み込み順で行う
- **データ**: localStorage キーはバージョン付き（`eiyokanri.*.v1`）。スキーマ変更時は移行関数を書いてキーを v2 に上げる
- **記録の意味**: entry は「摂取グラム」だけを持ち、栄養値は表示時に食材マスタから計算する。マスタの栄養値を修正すると過去の表示にも遡及する（意図した仕様）
- **DOM生成**: 新規・改修部分は `createElement` + `textContent`（innerHTML は使わない。セキュリティフック方針）
- **ロジック分離**: 計算・提案・換算などの純関数は将来 `logic/` 系ファイルに切り出し、Node で単体検証できる形を目指す
- **位置づけ**: 医療・栄養指導の代替ではなく家庭内の目安。UI上にも免責の一文を置く

## 既知の課題

- `AGE_TARGETS` のベース値は食事摂取基準2020と照合済み（2026-07-03、`source.url`/`referencedAt` 記載）。ただし離乳食比率（15/35/65/85%）は支援ガイドの進行イメージに基づく近似であり公式数表ではない
- ミルクの100mlあたり栄養は食品成分DB「乳児用調製粉乳」(13011) の13%調乳換算で照合済み。製品差があるためパッケージ表示優先（将来は製品値の設定機能も検討）
- `spoonGrams` の値が全食材で共通値（5g/15g）のまま。がゆ・ペースト等の実態値への精査が未実施（結線自体は完了）
- localStorage はオリジン単位。http(s) 配信へ移行する際は、移行前にエクスポート → 新オリジンでインポートすること（機能は実装済み）
- 自動テストがない（都度の Node アドホック検証のみ）
